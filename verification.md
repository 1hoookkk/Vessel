Project ARMAdillo: Z-Plane Emulation & Verification Guide1. Architectural OverviewThe E-mu Z-Plane filter (found in Morpheus, Ultra Proteus, Proteus 2000) is not a standard digital filter. It utilizes a proprietary interpolation method to morph between complex 14-pole transfer functions.The Core ComponentsThe H-Chip / FX8010: The DSP silicon. It operates on 32-bit fixed-point integers with a 67-bit accumulator.The Filter Cube: A 3D data structure allowing morphing across three axes:Morph (X): Interpolates between filter shapes (e.g., Vowel A to Vowel B).Freq (Y): Shifts the entire frequency set (Pitch Tracking).Transform (Z): Usually mapped to Velocity or Envelope amplitude.ARMAdillo Encoding: The system does not interpolate biquad coefficients ($a, b$) directly. It interpolates parameters $k_1$ (Frequency) and $k_2$ (Resonance) to ensure stability and musicality.2. Verification FrameworkPer the technical analysis, simply running the code is insufficient. You must verify the emulation against the "Digital Archaeology" standards.Protocol A: The "Null Filter" (F000) TestThe "Null Filter" is the identity state of the engine. It is a valid 14-pole configuration that results in a "Thru" signal.Setup: Load the parameters for Filter Index 0 (F000) from the extracted ROM data.Input: A Dirac Impulse (1.0 followed by 0.0s).Expected Output: A single sample of 1.0 followed by perfect silence.Failure Mode 1 (Ringing): $k_2$ resonance decoding is incorrect.Failure Mode 2 (Phase Shift): Zero placement ($a$ coefficients) is incorrect.Protocol B: The Linearity CheckARMAdillo ensures that linear changes in parameter space equal logarithmic changes in frequency space.Test: Sweep the k1 parameter linearly from 0.0 to 1.0.Measurement: Perform an FFT on the output.Verification: The Center Frequency ($F_c$) peak should move in Octaves.Correct: 100Hz -> 200Hz -> 400Hz -> 800Hz.Incorrect: 100Hz -> 200Hz -> 300Hz -> 400Hz (This indicates raw Hertz interpolation, which is wrong).Protocol C: Saturation AnalysisThe "warmth" of the E-mu filters comes from the specific way the FX8010 saturates the accumulator.Test: Drive the filter with a high-resonance setting and high input gain (> 0dB).Observation: The output should clip hard at INT_MAX (2147483647).Verification: Instead of wrapping around (digital glitching) or floating-point infinite growth, the signal should square off, adding odd harmonics. This acts as a built-in waveshaper.3. Implementation NotesThe provided C++ core (emu_zplane_core.cpp) implements:Bit-Accurate Math: Uses int32_t and int64_t to mimic the hardware registers.Trilinear Interpolation: Implements the 8-corner weighting logic.Coefficient Decoding: Includes the mapping logic for $k_1 \to \theta$ and $k_2 \to R$.Note on ROM Extraction:To fully utilize this engine, you must extract the "Filter Cube" presets from a real E-mu ROM (e.g., Proteus 2000).
